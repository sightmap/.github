# Getting help

Where to go depends on what you need.

## Questions about the spec or `.sightmap/` format

→ Discussions on **[sightmap/spec](https://github.com/sightmap/spec/discussions)** — that's where the format itself is defined.

## Bugs in an official SDK

Open an issue on the SDK repo:

- **JavaScript / TypeScript** — [sightmap/sightmap-js](https://github.com/sightmap/sightmap-js/issues)
- **Python** — [sightmap/sightmap-python](https://github.com/sightmap/sightmap-python/issues)

## Bugs or ambiguities in the spec itself

→ [sightmap/spec/issues](https://github.com/sightmap/spec/issues)

## Proposing a change to the spec

→ Start a Discussion on `sightmap/spec`. If it gains traction, follow the [SEP process](https://github.com/sightmap/spec/blob/main/seps/README.md).

## Security issues

→ See [SECURITY.md](SECURITY.md). Do not file these as public issues.

## Code of Conduct concerns

→ Email **subtext@fullstory.com**. See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

## Subtext product support

This is the open-source Sightmap project. If you're a Subtext customer, contact Subtext through your normal channel or visit [subtext.fullstory.com](https://subtext.fullstory.com).

## Response times

Best-effort. Maintainers aim for a first response on issues and PRs within **3 business days**. Discussions have no SLA — we read them all but reply when we can.
